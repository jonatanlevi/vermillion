## 2. מסלול המשתמש (מוצר)

### 2.1 הרשמה

1. התחברות (למשל Google OAuth).
2. השלמת פרופיל / onboarding.
3. **קביעת DNA יומי** — בחירת שעה (למשל 08:00) על השעון.
4. מרגע זה — **DNA יומי לא ניתן לשינוי** (אפליקציה + מסד נתונים).

### 2.2 ימי א׳–ה׳ (ותחרות)

1. המשתמש מגיע לשעת ה-DNA (לפי שעון מקומי).
2. משחק משחק אחד לפחות.
3. המערכת מנפיקה **Token**.
4. המשתמש לוחץ **חתום** — השרת מחשב ניקוד לפי קרבה לשעת DNA.
5. תוצאה נשמרת ב-`daily_stamps` (לוח תחרות).

| כלל | ערך |
|-----|-----|
| טווח חתימה מותר | 00:01 – 15:30 (שעון מקומי) |
| DNA יומי | **לא פעיל** |
| בחירת שעה | המשתמש בוחר שעה **בתוך הטווח** (למשל 10:00) |
| נעילה | לאחר קביעה ראשונה — **DNA שישי לצמיתות** |
| ניקוד | קרבה לשעה שבחר, בתוך החלון |

### 2.4 יום שבת — סיום האתגר

| כלל | ערך |
|-----|-----|
| טווח חתימה מותר | 21:00 – 23:59 (שעון מקומי) |
| DNA יומי | **לא פעיל** |
| בחירת שעה | בתוך הטווח (למשל 21:30) |
| נעילה | לאחר קביעה ראשונה — **DNA שבת לצמיתות** |
| ניקוד | קרבה לשעה שבחר, בתוך החלון |

### 2.5 יום ראשון

- DNA שישי/שבת נשמרים לשבועות הבאים (אותה שעה בכל שישי/שבת).

## 3. מנגנון Anti-Cheat (3 שכבות)

```
[משחק] → game-complete (אימות + Token) → [חתום] → stamp (שריפת Token + ניקוד + daily_stamps)
```

### שכבה 1 — `game-complete` (שרת)

| בדיקה | תוצאה אם נכשל |
|--------|----------------|
| JWT משתמש תקף | 401 Unauthorized |
| `APP_SECRET` בכותרת | 403 Forbidden |
| משך משחק ≥ מינימום למשחק | `GAME_TOO_FAST` |
| `started_at` לא עתידי; גיל session ≤ 30 דק׳ | `SESSION_EXPIRED` / `INVALID_SESSION_TIME` |
| `duration_ms` תואם שעון קיר | `DURATION_MISMATCH` |
| Upsert `game_sessions` עם Token חדש | Token ישן מתבטל באותו יום |

### שכבה 2 — `stamp` (שרת)

| בדיקה | תוצאה אם נכשל |
|--------|----------------|
| JWT + `APP_SECRET` | 401 / 403 |
| Token קיים, לא נוצל, שייך למשתמש/יום/חודש | `INVALID_TOKEN` |
| יום שישי/שבת: שעה בתוך חלון | `WINDOW_CLOSED` |
| שישי/שבת: יעד (DNA אתגר) הוגדר | `CHALLENGE_TIME_REQUIRED` |
| ימי חול: DNA מההרשמה | ניקוד מ-DNA |
| שריפת Token אטומית | מניעת חתימה כפולה |
| כשל כתיבה ל-`daily_stamps` | Rollback Token |

### שכבה 3 — לקוח

| כלל | מימוש |
|-----|--------|
| אין חתימה מוצלחת מקומית אם השרת נכשל | `saveGameStamp` → `ok: false` |
| חלון שישי/שבת לפי שעון מכשיר | `getDeviceTimezone()` |
| אין חתימה מחוץ לחלון / לפני שעת יעד | UI + בדיקה לפני invoke |

### נעילת זמנים (מסד נתונים)

טריגר `commitment_immutability_guard` (מיגרציה `20260519`):

- לא ניתן לעדכן `committed_hour/minute` לאחר קביעה.
- לא ניתן לעדכן `friday_target_*` / `saturday_target_*` לאחר קביעה.
- חל גם על עדכון ישיר דרך API (לא רק דרך האפליקציה).

## 4. נתונים אישיים וזמן

- **Timezone** מתעדכן ממכשיר המשתמש (`client_timezone`) לצורך חישוב יום ושעה.
- **לא** מוטמע שעון ישראל לכל המשתמשים.
- דוגמה: משתמש שנרשם בישראל וחותם בניו יורק — החתימה נמדדת לפי **שעון ניו יורק** ברגע הלחיצה.

| טבלה | תפקיד | גישת משתמש |
|------|--------|------------|
| `profiles` | פרופיל, timezone | קריאה/עדכון עצמי |
